# Тест-кейсы

## TC-01: выбор пользователя

**Шаги:**
1. Открыть frontend.
2. Выбрать другого пользователя в dropdown.

**Ожидаемо:**
- обновляются имя, сегмент, выгода, прогноз, офферы и cross-sell.

## TC-02: офферы соответствуют сегменту

**Шаги:**
1. Выбрать пользователя LOW.
2. Проверить офферы.
3. Выбрать пользователя HIGH.
4. Проверить офферы.

**Ожидаемо:**
- для каждого пользователя показываются офферы только его сегмента.

## TC-03: прогноз отображается по каждой валюте

**Шаги:**
1. Выбрать пользователя с несколькими счетами.
2. Открыть блок «Прогноз».

**Ожидаемо:**
- каждая валюта лояльности имеет прогноз на месяц и год.

## TC-04: backend возвращает 404

**Шаги:**
1. Вызвать `/api/users/999999/loyalty-summary`.

**Ожидаемо:**
- API возвращает HTTP 404.

## TC-05: frontend работает без backend

**Шаги:**
1. Не запускать API.
2. Открыть `frontend/index.html`.

**Ожидаемо:**
- открывается fallback-демо на sample JSON.

## TC-06: тёмная тема

**Шаги:**
1. Нажать кнопку темы.

**Ожидаемо:**
- интерфейс переключается на тёмную тему без перезагрузки.

## TC-07: мобильная адаптивность

**Шаги:**
1. Открыть DevTools.
2. Выставить ширину 375 px.

**Ожидаемо:**
- карточки становятся в одну колонку, текст не обрезается.
